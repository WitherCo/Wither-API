API Usage Policy

Last Updated: May 7, 2025

1. Introduction

This API Usage Policy governs your use of the Wither API Gateway. By using our API, you agree to comply with this Policy, our Terms of Service, and Privacy Policy.

Our goal is to provide a reliable, secure, and efficient API that helps you integrate with various services. This Policy is designed to ensure fair usage and maintain the quality of service for all users.

2. API Authentication

All API requests must be authenticated using API keys. Each API key is associated with a specific user account and has certain permissions and rate limits.

To authenticate your requests, include your API key in the HTTP header:

X-API-Key: your_api_key_here

API keys should be kept secure and never shared publicly. Do not include API keys in client-side code or repositories. We recommend storing your API keys in environment variables or a secure secret management system.

Important Security Notice: If you believe your API key has been compromised, immediately revoke it and generate a new one. You can manage your API keys through your account dashboard.

3. Rate Limits

To ensure fair usage and service stability, we implement rate limiting on API requests. The default rate limit is 100 requests per hour per API key. Different endpoints may have different rate limits based on their resource intensity.

When you exceed your rate limit, the API will return a 429 Too Many Requests response with a Retry-After header indicating when you can resume making requests.

Rate limits are subject to change based on service load and other factors. If you need higher rate limits for your application, please contact us to discuss your requirements.

Endpoint Category Rate Limit Notes
General API 100 requests/hour Default limit for most endpoints
Webhook Delivery 50 requests/minute Higher frequency allowed for event processing
Email Sending 20 requests/minute To prevent spam and abuse

4. Data Usage and Storage

Any data you provide through the API should comply with our Privacy Policy and applicable data protection laws. You are responsible for obtaining necessary consents for the collection, use, and processing of any personal data.

We may store certain data transmitted through the API for debugging, analytics, and improving our services. Request logs are typically retained for 30 days. Sensitive data such as authentication credentials or personal information should never be included in URLs or query parameters.

5. API Versioning and Changes

We may update or modify the API from time to time. Major changes that could break existing integrations will be released as new API versions. We strive to maintain backward compatibility within the same API version.

We will provide advance notice of significant changes through our documentation, email notifications, and the API changelog. We recommend subscribing to our updates to stay informed about changes that may affect your integration.

6. Webhooks

Our webhook system allows you to receive real-time notifications about specific events. When setting up webhook endpoints, ensure they:

  • Respond quickly (within 5 seconds) to webhook deliveries
  • Return appropriate HTTP status codes (2xx for success)
  • Implement proper security measures, such as verifying webhook signatures
  • Are publicly accessible and properly configured to handle incoming requests

We may disable webhook deliveries if your endpoint consistently fails to receive them properly. Webhook events are retried up to 3 times with exponential backoff in case of delivery failures.

7. Error Handling

The API uses standard HTTP status codes to indicate the success or failure of requests. Here are some common status codes you may encounter:

  • 200 OK: Request succeeded
  • 201 Created: Resource was successfully created
  • 400 Bad Request: Request was invalid or could not be processed
  • 401 Unauthorized: Authentication failed or was not provided
  • 403 Forbidden: Authentication succeeded but the user lacks permission
  • 404 Not Found: Requested resource does not exist
  • 429 Too Many Requests: Rate limit exceeded
  • 500 Internal Server Error: An error occurred on the server

Error responses include a JSON body with more details about the error:

{ "error": { "code": "invalid_request", "message": "The request was invalid", "details": "The required field 'name' is missing" } }

Implement proper error handling in your applications to gracefully handle API errors and rate limits.

8. Acceptable Use

When using our API, you must comply with all applicable laws and regulations. The following activities are strictly prohibited:

  • Scraping, crawling, or downloading large amounts of data for purposes other than integration
  • Attempting to bypass or circumvent API rate limits or security measures
  • Using the API to send spam, unsolicited messages, or harmful content
  • Reverse engineering the API or attempting to access undocumented features
  • Using the API in a way that could degrade its performance or availability for others
  • Building applications that violate our Terms of Service or Privacy Policy

We reserve the right to suspend or terminate API access for users who violate this Policy or our Terms of Service.

9. Support and Maintenance

We strive to maintain high availability for our API. However, occasional maintenance windows and service interruptions may occur. We will provide advance notice of scheduled maintenance whenever possible.

For API support, you can:

  • Submit issues or questions through our GitHub repository
  • Contact us at support@witherco.xyz for urgent matters
  • Refer to the API documentation for implementation guidance

We may occasionally deprecate API features or endpoints that are no longer supported. Deprecated features will be clearly marked in the documentation with information about alternatives and timelines for removal.

10. Monitoring and Compliance

We monitor API usage to ensure compliance with this Policy. We may contact you if we detect unusual patterns or potential violations. In cases of severe or repeated violations, we may take action to protect our systems and other users, including temporarily or permanently suspending your API access.

We welcome feedback on this Policy and our API services. If you have suggestions for improvements or questions about compliance, please contact us.
